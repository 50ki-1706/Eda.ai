## 概要

実装完了後に必須の確認作業を行うskill。コードベースへの変更が正しく反映されていることを保証する。

## 実行タイミング

コードベースへの変更完了後、**必ず**実行する。これは省略可能な推奨事項ではなく、実装プロセスの一部として必須である。

## 必須確認手順

以下の3つのコマンドをこの順序で実行する。いずれもスキップ不可。

1. `pnpm typecheck`
   TypeScript の型エラーがないことを確認する。

2. `pnpm check`
   Biome のフォーマット・Lint エラーがないことを確認する。`pnpm check` は format と lint の両方をカバーする。

3. `pnpm build`
   本番ビルドが通ることを確認する。`prisma generate` も自動実行される。

## 状況に応じた追加確認

必須確認に加え、変更内容に応じて以下も実施する。

- **UI 変更がある場合** → `pnpm dev` で該当ページの動作確認を行う。
- **Prisma / DB スキーマの変更がある場合** → `pnpm exec prisma generate` および `pnpm exec prisma migrate dev` で整合性を確認する。
- **関連するテストが存在する場合** → テストを実行する。

## ルール

- 3つの必須コマンドはスキップ不可。
- 実行順序は typecheck → check → build の固定順。
- いずれかが失敗したら即座に原因を確認し、修正後に再確認する。
- 全ての必須確認が成功するまで実装完了とは見なさない。
