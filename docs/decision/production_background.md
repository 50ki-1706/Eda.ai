# 制作背景

## AIチャットの課題
- AIチャットツールは一方向で会話が続いていて、例えばAIのレスポンスに対して、複数聞きたい内容がある場合に、まとめて聞いてしまうと、深掘りをした回答が聞けないことが多く、また別々で聞こうとすると、元のレスポンスが上に流れていってしまうということが、大学でICTの授業をとっていて、AIでリサーチしている時によくありました。
- このアプリケーションを提案した時に、周りの学生からは同様の意見が出ており、一定の需要があると考えました。
## Eda.aiの目的
- AIチャットで、ブランチに分散させて,それぞれのブランチで会話を続けて、またそのブランチを特定のブランチにマージして、チャットを使いやすくする。
## chatGPTのブランチ機能の課題
実はchatGPTには2025年9月にブランチ機能を出しています。
しかし、ブランチ機能の存在について、知っている人はあまり多くないと感じます。
理由は,チャットのレスポンスの下にある、その他のアクションから、新しいチャットに分岐ボタンをおさなければならず、まず見つけるのが難しい。
またこのブランチ機能を使うと新しいチャットを生成するのですが、グラフがUIとして表示されるのではなく、生成されるチャットのsummaryが"枝分かれ●{元のチャットのsummary}"という名前になるだけです。
また、このブランチ機能には各ブランチのマージ機能がないように見えます。マージ機能が使えると、ユーザとの対話で得た多角的な視点をAIが読み込むことができ、さらなる創造性を生み出せると感じています。

したがって、このEda.aiでは、ユーザーが直感的に使いやすいUI(グラフや動線)とマージ機能を持つことで、研究者やクリエイターなど、創造性をより必要とする人たちに価値が届けられると思っています。
